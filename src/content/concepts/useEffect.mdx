# useEffect Hook in React

The `useEffect` hook lets you perform side effects in your components. Side effects can include:

-   Data fetching
-   DOM manipulation
-   Subscriptions
-   Timer operations

## Basic Example

Here's a simple example that shows how to use it with a counter:

```jsx
import React, { useState, useEffect } from 'react';

const UseEffect = () => {
    const [count, setCount] = useState(0);

    useEffect(() => {
        document.title = `Count is ${count}`;
    }, [count]);

    return (
        <div className='text-center'>
            <p className='text-xl mb-2'>Count: {count}</p>
            <p className='text-sm text-gray-600 mb-4'>
                Check your browser tab title!
            </p>
            <button
                onClick={() => setCount((c) => c + 1)}
                className='bg-blue-500 text-white px-4 py-2 rounded hover:bg-blue-600'
            >
                Increment
            </button>
        </div>
    );
};

export default UseEffect;
```

## Understanding the Code

Let's break down the key elements:

1. **Effect Setup**

```jsx
useEffect(() => {
    document.title = `Count is ${count}`;
}, [count]);
```

-   The first argument is the effect function
-   The second argument `[count]` is the dependency array
-   The effect runs when `count` changes

## Common Use Cases

```jsx
// Example with cleanup
useEffect(() => {
    const subscription = someAPI.subscribe();

    // Cleanup function
    return () => {
        subscription.unsubscribe();
    };
}, []);

// Example with async operation
useEffect(() => {
    async function fetchData() {
        const response = await api.getData();
        // Handle response
    }
    fetchData();
}, []);
```

## Best Practices

-   Always specify dependencies array
-   Use cleanup function when needed
-   Avoid infinite loops
-   Keep effects focused on one responsibility
